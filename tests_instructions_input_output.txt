Каждый тест подразумевает единственное значение входных параметров.
Каждому тесту соответствует единственная последовательность Систем Счисления.
Первые 20 тестов выведут каждую возможную ошибку.
После тестов 22, 23, 25, 26, 27 находятся ПОЯСНЕНИЯ.

Особенности ввода:
1. Программа работает только с первыми двумя найденными числами!
(т.е. важна правильность ввода только до конца второго числа!)
2. Можно вводить числа вида 00000A (поддержка незначащих нулей).
3. Можно вводить отрицательные числа вида -A и -0000A.

Особенности вывода:
1. Над FINISHED: SUCCESS - найденный НОД!
2. WRONG SYSTEM - ошибки, связанные с входными параметрами (Системами Счисления)
3. WRONG INPUT - ошибки, связанные со структурной входимых чисел
4. ERROR - ошибки, связанные с вычислениями

Формат верного ввода:
1. Числа A и B разделены между собой, имеют перед собой, имеют за собой только пробельные символы, табуляции, перехоы на следующую строку, преждевременный конец ввода.
2. Числа A и B могут быть представлены в виде +A, -A, +B, -B, но не ---A, +-+B и т.п.
3. Числа A и B состоят только из верных цифр для заданной Системы Счисления.
4. На вход программе (в консоли) подаются ровно 3 числа из диапазона 2..70.
5. На одной строке с числом B не должны находиться лишние символы (при считывании с файла).
Во всех других случаях программа выдаст соответствующую ошибку.

Замечание: при вводе с файла подобного текста:
1234
1343
1555
будут считаны числа 1234 и 1343 в соответствующих СС, ведь программа считывает только первые два числа.

На протяжении всего ввода у пользователя есть возможность остановить ввод с помощью CTRL+D.

Удачные сообщения:
20-27. FINISHED: SUCCESS <- НОД найден!

Ошибки, связанные с арифметическими вычислениями:
19. ERROR: NUMBER C IS >2000 LENGTH LONG <- длина числа C превышает 2000;

Список ошибок, связанных с вводом:
1.  WRONG SYSTEM: LESS THAN THREE <- меньше трёх аргументов;
2.  WRONG SYSTEM: MORE THAN THREE <- больше трёх аргументов;
3.  WRONG SYSTEM: NOT A NUMBER <- хотя бы один аргумент не число;
4.  WRONG SYSTEM: OUT OF RANGE <- аргумент вне диапазона;
5.  WRONG INPUT: NOT ENOUGH NUMBERS <- введено меньше двух чисел;
6.  WRONG INPUT: 1ST NUMBER, INCORRECT SEQUENCE <- число A: неверная последовательность (найдено: ++ или +- или -+ или --);
7.  WRONG INPUT: 1ST NUMBER, INCORRECT SYMBOL <- во время поиска числа A найден запрещённый символ!;
8.  WRONG INPUT: 1ST NUMBER, LONE SIGN <- число A: одинокий знак (найден изолированный + или изолированный -);
9.  WRONG INPUT: 1ST NUMBER, INCORRECT DIGIT <- число A: цифра неверного вида;
10. WRONG INPUT: 1ST NUMBER, DIGIT IS NOT IN THE SYSTEM <- число A: цифра верная, но вне системы;
11. WRONG INPUT: 1ST NUMBER, LENGTH >1000 <- число A: длина числа >1000;
12. WRONG INPUT: ONLY ONE NUMBER <- введено только одно число!;
13. WRONG INPUT: 2ST NUMBER, INCORRECT SEQUENCE <- число B: неверная последовательность (найдено: ++ или +- или -+ или --);
14. WRONG INPUT: 2ST NUMBER, INCORRECT SYMBOL <- во время поиска числа B найден запрещённый символ!;
15. WRONG INPUT: 2ST NUMBER, LONE SIGN <- число B: одинокий знак (найден изолированный + или изолированный -);
16. WRONG INPUT: 2ST NUMBER, INCORRECT DIGIT <- число B: цифра неверного вида;
17. WRONG INPUT: 2ST NUMBER, DIGIT IS NOT IN THE SYSTEM <- число B: цифра верная, но вне системы;
18. WRONG INPUT: 2ST NUMBER, LENGTH >1000 <- число B: длина числа >1000;
